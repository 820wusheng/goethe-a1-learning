# 📚 Skill使用指南

> 如何使用`goethe-a1-builder` Skill开发新功能或修复问题

---

## 🎯 什么是Skill？

Skill是一个**强制执行开发流程**的工具，确保：
- ✅ 开发前必读踩坑文档
- ✅ 开发中对照检查清单
- ✅ 开发后逐项Review
- ✅ 浏览器完整测试
- ✅ 不重复历史错误

**核心原则**: 写了流程就必须遵守，不能跳过任何步骤！

---

## 🚀 快速开始

### 调用Skill

在Claude Code中输入：

```bash
/goethe-a1-builder
```

或者在对话中说：

```
帮我用goethe-a1-builder添加新功能
```

### Skill自动执行

1. **阶段1: 开发前准备**
   - 自动读取`PITFALLS.md`
   - 创建TODO检查清单
   - 列出可能踩的坑

2. **阶段2: 数据解析**
   - 从PDF提取内容
   - 人工校对翻译（不用自动API）
   - 验证关键词

3. **阶段3: 页面开发**
   - 使用本地音频文件
   - 实现答题功能
   - 相对路径引用

4. **阶段4: 开发后Review**
   - 重新读`PITFALLS.md`
   - 运行自动检查脚本
   - 逐项确认清单

5. **阶段5: 浏览器测试**
   - 启动HTTP服务器
   - 逐页点击测试
   - 验证所有功能

6. **阶段6: 提交文档**
   - 三次确认
   - Git提交
   - 更新文档

## 📋 强制检查清单（25项）

Skill会自动对照这些检查点：

### 开发前（4项）
- [ ] 已读PITFALLS.md全文
- [ ] 已创建TODO检查清单
- [ ] 已列出本次可能踩的坑
- [ ] 已确认避免措施

### 开发中（4项）
- [ ] 音频使用本地文件
- [ ] 翻译手动完成
- [ ] 对照PDF校对
- [ ] 相对路径引用

### 开发后（5项）
- [ ] 重新读PITFALLS.md
- [ ] 运行自动检查脚本
- [ ] grep验证关键词
- [ ] 检查音频文件存在
- [ ] 检查所有路径正确

### 浏览器测试（8项）
- [ ] 启动HTTP服务器
- [ ] 打开主页测试
- [ ] 打开每套试卷
- [ ] 点击每个音频播放器
- [ ] 点击翻译按钮
- [ ] 测试答题流程
- [ ] 检查控制台无错误
- [ ] 检查无404错误

### 提交前（4项）
- [ ] 所有TODO标记completed
- [ ] 三次确认
- [ ] Git commit包含检查点
- [ ] PITFALLS.md已更新

## 🔴 历史踩坑（必须避免）

### 元坑：不遵守流程
**最严重的错误！**
- ❌ 写了"必读PITFALLS"但不读
- ❌ 写了"必须测试"但不测
- ✅ Skill强制执行，不能跳过

### 1. 音频播放
- ❌ 使用外部链接（会失效）
- ❌ target="_blank"跳转新窗口
- ✅ 使用本地mp4文件
- ✅ `<audio controls>`在页面内播放

### 2. 翻译准确
- ❌ 使用自动翻译API
- ❌ 人名翻译错误（Schneider→裁缝）
- ✅ 人工翻译所有内容
- ✅ 对照PDF官方文档

### 3. 页面完整
- ❌ 只做部分功能
- ❌ 按钮点不了
- ✅ 4部分齐全（听说读写）
- ✅ 所有功能测试通过

### 4. 答题功能
- ❌ 提交无反馈
- ❌ 不显示对错
- ✅ 完整流程（选择→提交→对错→分数）
- ✅ 每个选项可点击

### 5. 文件路径
- ❌ 使用绝对路径
- ❌ 重构后路径失效
- ✅ 使用相对路径
- ✅ 目录结构清晰

### 6. 数据完整
- ❌ 简化内容
- ❌ 跳过部分题目
- ✅ 对照PDF完整保留
- ✅ 验证题目数量

### 7. 自测充分
- ❌ 只运行代码
- ❌ 不浏览器测试
- ✅ 逐页点击验证
- ✅ 所有功能实测

## 🔧 手动执行Skill流程

如果不用Skill自动执行，可以手动按照流程：

### 1. 开发前

```bash
cd /path/to/goethe-a1-exam

# 阅读踩坑文档
cat PITFALLS.md

# 创建检查清单
# 在纸上或TODO中列出本次要避免的坑
```

### 2. 开发中

```bash
# 确保使用本地文件
ls audio/*.mp4  # 检查音频存在

# 人工翻译
# 不使用自动API，对照PDF手动翻译

# 相对路径
# 使用../data/xxx.json, ../audio/xxx.mp4
```

### 3. 开发后

```bash
# 重新读文档
cat PITFALLS.md

# 自动检查
echo "target='_blank': $(grep -r "target=\"_blank\"" html/*.html | wc -l)"
echo "施奈德: $(grep -c "施奈德" data/*.json)"
echo "19.95欧元: $(grep -c "19.*95.*欧元" data/*.json)"
```

### 4. 浏览器测试

```bash
# 启动服务器
python3 -m http.server 8080

# 打开浏览器
# http://localhost:8080/html/index-main.html

# 逐页测试
# - 点击所有链接
# - 播放所有音频
# - 测试所有功能
```

### 5. 提交

```bash
# 三次确认
echo "1. 读了PITFALLS.md? yes"
echo "2. 浏览器测试? yes"
echo "3. 音频能播放? yes"

# 提交
git add -A
git commit -m "feat: xxx

完成:
- ✅ 具体内容
- ✅ 已读PITFALLS.md
- ✅ 浏览器测试通过

检查点:
- ✅ 音频: 本地文件
- ✅ 翻译: 人工翻译
- ✅ 测试: 浏览器实测
"
git push
```
